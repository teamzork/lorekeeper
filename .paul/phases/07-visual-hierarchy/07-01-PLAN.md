<objective>
## Goal
Create an interactive visual relationship graph showing how entities connect within a world — nodes represent entities (styled by type), edges represent relationships (labeled, directed). Users can pan, zoom, drag nodes, and click to navigate to entity details.

## Purpose
The graph gives worldbuilders a bird's-eye view of how their universe fits together — visually revealing clusters, orphans, and connection patterns that are invisible in list views. This is one of Lorekeeper's most visually distinctive features.

## Output
- `app/(app)/worlds/[worldId]/graph/page.tsx` — Graph tab route page
- `components/worlds/graph/WorldGraph.tsx` — Main graph container
- `components/worlds/graph/EntityNode.tsx` — Custom React Flow node per entity type
- `components/worlds/graph/RelationshipEdge.tsx` — Custom labeled edge
- Updated `EntityNav.tsx` with Graph tab
- `@xyflow/react` and `@dagrejs/dagre` installed
</objective>

  <action>
    **EntityNode.tsx — Custom node component:**
    - Receives entity data via `data` prop: { name, entityType, entityId }
    - Renders a rounded card with:
      - Type-specific icon from lucide-react (User for character, MapPin for place, Shield for faction, Gem for artifact, BookOpen for lore)
      - Type-specific accent color on the left border:
        - character: purple (hsl(262 83% 68%) — primary)
        - place: emerald (hsl(160 60% 45%))
        - faction: amber (hsl(45 90% 65%) — accent)
        - artifact: cyan (hsl(190 80% 55%))
        - lore: rose (hsl(350 70% 60%))
      - Entity name in semibold
      - Type label in muted text below name
    - Background: card color (hsl(224 71% 6%))
    - Has a source Handle at bottom and target Handle at top (for edges)
    - Cursor: pointer (clickable)

    **RelationshipEdge.tsx — Custom edge component:**
    - Uses React Flow's `EdgeLabelRenderer` + `BaseEdge`
    - Renders the relationship label centered on the edge path
    - Label styled: small text, muted-foreground, semi-transparent background pill
    - Edge color: muted border color
    - Animated: false (keep it clean)
    - Marker end: arrow

    **WorldGraph.tsx — Main graph container:**
    - Accepts `worldId` prop
    - Uses `useQuery(api.entities.list, ...)` for each entity type to load all entities
    - Uses `useQuery(api.relationships.listForWorld, { worldId })` to load all relationships
    - Converts entities to React Flow nodes:
      - Each entity becomes a node with `id: entity._id`, `type: "entity"`, `data: { name, entityType, entityId: entity._id }`
    - Converts relationships to React Flow edges:
      - Each relationship becomes an edge with `source: rel.fromId`, `target: rel.toId`, `type: "relationship"`, `data: { label: rel.label }`
    - Computes layout using dagre:
      - Create dagre graph, set graph direction TB (top-to-bottom) or LR (left-to-right)
      - Set default node width ~200, height ~70
      - Add nodes and edges, run dagre.layout()
      - Map dagre positions back to React Flow nodes
    - Renders `<ReactFlow>` with:
      - `nodeTypes={{ entity: EntityNode }}`
      - `edgeTypes={{ relationship: RelationshipEdge }}`
      - `fitView` prop to auto-fit on load
      - Dark background: `<Background variant="dots" color="hsl(223, 47%, 15%)" gap={20} />`
      - `<Controls />` for zoom controls (styled to match dark theme)
      - `<MiniMap />` optional — include if it looks good
    - On node click: navigate to `/worlds/${worldId}/entities/${node.data.entityId}?type=${node.data.entityType}`
    - Import React Flow CSS: `import '@xyflow/react/dist/style.css'`
    - Override React Flow default styles with dark theme using a wrapper div with Tailwind classes

    **Empty state handling in WorldGraph:**
    - If all entity queries return empty arrays: show empty state with Network icon + "No entities yet" + "Add characters, places, and other entities to see them visualized here"
    - If entities exist but no relationships: show the nodes in a grid layout (no dagre needed) with a hint: "Add relationships between entities to see connections"

    **graph/page.tsx — Route page:**
    - "use client" directive
    - Get worldId from useParams()
    - Render `<WorldGraph worldId={worldId} />`
    - Full height: the graph should fill available space. Use `min-h-[600px]` or `h-[calc(100vh-300px)]` to give it room
