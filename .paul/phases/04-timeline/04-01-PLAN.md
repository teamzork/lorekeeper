<objective>
## Goal
Create the events data model and full CRUD UI for timeline events within a world — list, create, edit, delete, and detail views.

## Purpose
Timeline events are the temporal backbone of a world. Writers need to record what happened, when, and see events in chronological order. This plan delivers the data layer and all CRUD interactions; Plan 02 will add entity linking and visual chronology.

## Output
- `events` Convex table with CRUD mutations/queries
- Timeline tab in world navigation
- Timeline list page (chronological)
- Event detail page with edit/delete
- Create/Edit/Delete dialogs following existing patterns
</objective>

  <name>Task 1: Events schema and Convex CRUD</name>
  <files>convex/schema.ts, convex/events.ts</files>
  <action>
    1. Add `events` table to schema.ts:
       - worldId: v.id("worlds")
       - name: v.string()
       - description: v.optional(v.string())
       - date: v.string() — free-form in-world date (e.g. "Year 3, Age of Fire", "1342 AD", "Day 7")
       - era: v.optional(v.string()) — grouping label (e.g. "First Age", "Pre-War")
       - significance: v.optional(v.string()) — why this event matters
       - createdBy: v.id("users")
       - createdAt: v.number()
       - updatedAt: v.number()
       - Index: by_world on worldId

    2. Create convex/events.ts with these functions (follow entities.ts patterns):
       - list(worldId): query, returns events sorted by _creationTime (ascending). Check membership.
       - get(id): query, returns single event. Check membership.
       - count(worldId): query, returns event count for world overview.
       - create(worldId, name, description?, date, era?, significance?): mutation. Check canWrite. Set createdBy, createdAt, updatedAt.
       - update(id, name?, description?, date?, era?, significance?): mutation. Check canWrite. Set updatedAt.
       - remove(id): mutation. Check canWrite. Delete event.

    Use the same checkMembership/canWrite pattern from entities.ts. Import and reuse if exported, otherwise duplicate the helper.

    Note on date sorting: Since dates are free-form fantasy strings, sort by _creationTime for now. Plan 02 may add a sortOrder field for manual ordering.
  </action>
  <verify>
    - npx convex dev should push schema without errors
    - Convex dashboard shows events table
    - TypeScript compiles with no errors
  </verify>
